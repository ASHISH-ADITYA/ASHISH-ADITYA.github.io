Ethical Hacking & Penetration Testing

C-DAC Noida — Training / Internship

I completed an intensive training and internship program in Ethical Hacking and Penetration Testing at the Centre for Development of Advanced Computing (C-DAC), Noida. The program was structured around building practical, hands-on competence in offensive security techniques used in professional security assessments.

The curriculum covered core areas including network reconnaissance and enumeration, vulnerability identification and assessment, exploitation methodologies, post-exploitation techniques, and reporting standards used in professional penetration testing engagements. Training was conducted in controlled lab environments that simulated real-world network and application architectures.

This experience gave me a structured understanding of how attackers approach systems — which directly informs how I think about building secure software. It also reinforced the importance of systematic documentation when reporting findings to development and operations teams.

Android Security Analysis with MobSF

MNIT Jaipur — ISEA-III Initiative (Virtual Internship)

As part of the ISEA Phase-III (Information Security Education and Awareness) initiative, I worked as a virtual intern under MNIT Jaipur on the analysis of Mobile Security Framework (MobSF) for Android application security. The objective was to evaluate how effectively MobSF identifies security weaknesses in Android APKs through automated static analysis.

My work involved running static analysis scans on multiple Android application packages and systematically reviewing the output. I categorized over 20 distinct vulnerability types across the scanned APKs — including insecure manifest configurations, overly permissive access controls, hardcoded credentials, exposed API endpoints, and misconfigured content providers that could serve as attack surfaces in production environments.

Beyond running scans, I focused on understanding the gap between what automated tools flag and what constitutes an actual exploitable risk. This distinction is critical in security work — tool output alone does not equal a finding. I documented each vulnerability with context on its severity, exploitability, and recommended remediation steps, building a structured approach to translating scanner results into security reports that engineering teams can act on.

Visual Surveillance & Anomaly Detection for Women Safety

IIT Bhubaneswar — Research Internship

I interned at IIT Bhubaneswar on the research project titled "Visual Surveillance and Extreme Anomaly Detection for Women Safety." The project aimed to develop robust anomaly detection capabilities for surveillance systems, with a specific focus on identifying threatening or dangerous situations in video feeds to improve public safety for women.

My primary responsibility was supporting the data pipeline that fed into vision-language model workflows. This involved collecting anomalous scene videos from diverse surveillance contexts, creating detailed textual annotations for each video, and ensuring that visual evidence was accurately aligned with descriptive labels suitable for downstream model training and evaluation. I curated and annotated more than 180 anomalous video samples, which became part of the dataset used for training and benchmarking the detection models.

The work required careful attention to annotation consistency and labeling standards — small inconsistencies in how scenes are described can introduce noise that degrades model performance, especially in safety-critical applications where false negatives carry real consequences. This internship gave me firsthand exposure to how data quality at the annotation stage directly determines the reliability of the models built on top of it.
